本文档基于 [pc_architecture_design.md](file:///Users/wenguanggu/MyProjects/Python/WebVideoDownloader/pc_architecture_design.md) 输出第一版界面设计方案，目标是为当前项目的 PC 端视频下载工具提供一套可直接用于产品评审和前端实现的 UI/UX 设计说明。

## 2. 设计目标

第一版界面设计重点解决以下问题：

- 用户打开软件后能立即理解“可以粘贴链接开始下载”
- 核心操作路径最短：输入 URL、选择清晰度、选择输出目录、开始下载
- 下载任务状态高信息密度但不混乱
- 下载失败时可以快速定位原因并恢复
- 页面结构能够贴合 `Tauri + React + TypeScript + Python Downloader Engine` 的实现方式

第一版优先级聚焦以下三件事：

1. 快速发起下载
2. 清楚管理任务
3. 遇到问题可定位并恢复

---

## 3. 设计基线

### 3.1 产品定位

该产品是一个 `PC 优先`、`工具型`、`任务驱动` 的桌面视频下载工具。

### 3.2 当前阶段判断

当前界面方案默认产品阶段为：

- `MVP / Beta 早期`

### 3.3 设计原则

- 首页必须聚焦主任务，而不是承载所有信息
- 任务页必须支持高频管理操作
- 设置页必须服务稳定运行，而不是配置堆砌
- 诊断页必须服务真实排障，而不是展示装饰性系统信息
- UI 层不暴露下载引擎和命令行实现细节

---

## 4. 信息架构

第一版建议采用 `左侧导航 + 右侧主工作区` 的典型桌面布局。

### 4.1 一级页面

- `新建下载`
- `任务管理`
- `设置`
- `诊断与日志`
- `关于与合规`

### 4.2 页面职责划分

| 页面 | 目标 | 优先级 |
| --- | --- | --- |
| `新建下载` | 快速创建下载任务 | 高 |
| `任务管理` | 管理所有任务、查看状态和处理失败 | 高 |
| `设置` | 管理默认行为与运行配置 | 中高 |
| `诊断与日志` | 排查环境与运行问题 | 高 |
| `关于与合规` | 承载版本、协议、说明和反馈入口 | 中 |

### 4.3 为什么这样分

- `新建下载` 保证核心操作路径最短
- `任务管理` 承载高信息密度任务列表，避免首页过载
- `设置` 和 `诊断` 分开，避免将“配置”与“故障排查”混杂
- `关于与合规` 单独放置，便于正式发布时补齐法律与版本说明

## 5. 全局布局设计

### 5.1 窗口建议

- 最小窗口尺寸：`1180 x 760`
- 左侧导航宽度：`220px`

### 5.2 全局结构

- 左侧：一级导航
- 顶部：页面标题、运行时全局状态、日志快捷入口
- 主内容区：页面主体

```text

│ Logo / 产品名                    运行时状态: 正常 / 警告     日志   帮助     │
├───────────────┬──────────────────────────────────────────────────────────────┤

└───────────────┴──────────────────────────────────────────────────────────────┘
```

### 5.4 布局原则

- 保持工具型桌面软件的信息密度

## 6. 页面一：新建下载

### 6.1 页面目标

让用户用最短路径完成一次下载任务创建。

### 6.2 页面结构

- 顶部：标题 + 一句话说明
- 左侧：下载表单区
- 右侧：运行时状态 + 最近任务摘要
- 底部：最近任务列表

### 6.3 页面线框

```text
┌──────────────────────────────────────────────────────────────────────┐

│ 粘贴视频链接，选择参数后立即开始下载                                │
├───────────────────────────────┬──────────────────────────────────────┤
│ 视频链接                       │ 运行时状态                           │
│ [ 输入框............................................... ] [粘贴]    │

│ 清晰度                         │ yt-dlp    正常 / 缺失                │
│ ( ) 最佳  ( ) 1080p  ( ) 720p │ ffmpeg    正常 / 缺失                │
│ ( ) 480p                      │ ffprobe   正常 / 缺失                │
│                               │ [打开诊断页]                         │
│ 输出目录                       │                                      │
│ [ /Users/.../Downloads ] [选择] [打开]                               │
│                               │ 最近任务摘要                         │
│ 文件命名规则                   │ 进行中 2  已完成 18  失败 1          │
```

### 6.4 核心组件

- `UrlInputPanel`
- `QualitySelector`
- `OutputDirectoryPicker`
- `DownloadOptionsPanel`
- `RuntimeHealthCard`
- `TaskSummaryCard`
- `RecentTasksList`
- `PrimaryActionBar`

### 6.5 核心交互

- 打开应用时，焦点默认落在 `视频链接输入框`
- 支持 `Ctrl/Cmd + V` 直接粘贴
- 当剪贴板内容是 URL 时，可展示 `检测到剪贴板链接，点击填入`
- 点击 `开始下载` 前执行以下校验：
  - URL 不能为空
  - URL 格式必须基本合法
  - 输出目录必须可写
  - 运行时依赖必须满足最低要求
- 创建任务成功后：
  - 显示顶部轻提示 `任务已加入下载队列`
  - 最近任务列表插入新任务
  - 可选清空表单

### 6.6 页面状态

- 空态：输入框为空，右侧显示简短引导
- 加载态：按钮变为 `创建任务中...`
- 成功态：顶部轻提示，不中断操作
- 错误态：在表单区域内给出结构化错误反馈
- 禁用态：依赖缺失时禁用 `开始下载`

### 6.7 异常反馈文案

- `请输入视频链接`
- `当前内容不像一个有效的视频链接`
- `所选目录不可写，请重新选择`
- `下载引擎不可用，请先完成运行时修复`
- `当前链接暂时无法解析，建议稍后重试或查看日志`

### 7.2 页面结构

### 7.4 表格字段建议

- 状态
- 标题
- 来源站点
- 进度
- 下载速度
- 剩余时间
- 清晰度
- 输出目录
- 更新时间
- 操作

### 7.5 桌面端高效交互

- 支持右键菜单：
  - `重试`
  - `取消`
  - `打开文件`
  - `打开目录`
  - `复制原始链接`
  - `复制错误信息`
- 双击任务行打开详情抽屉
- 失败任务高亮状态，但避免全屏高对比红色噪音

### 7.6 状态设计

- 下载中：显示进度条、速度、剩余时间
- 排队中：显示 `等待执行`
- 已完成：主操作为 `打开文件`
- 失败：显示错误摘要与 `重试`
- 已取消：显示 `已取消` 与 `重新下载`

### 7.7 异常恢复

每个失败任务至少提供一个恢复动作：

- 运行时错误：跳转诊断页
- 路径错误：重新选择目录
- 链接错误：复制链接重新编辑
- 站点限制：查看完整日志

## 11. 首页与任务页的关系

第一版不建议在首页中直接嵌套完整任务表。

### 原则

- 首页只保留 `最近任务摘要`
- 完整任务管理统一放在 `任务管理` 页面

### 原因

- 避免首页过载
- 保证首页聚焦主任务
- 让任务列表页承担高频管理职责

## 13. 视觉层级建议

- 一级操作：`开始下载`
- 二级操作：`选择目录`、`打开目录`、`重试`
- 次级信息：速度、剩余时间、路径
- 警告信息：依赖缺失、站点限制、权限问题
- 危险动作：删除记录、重置设置

原则：

- 错误信息只在必要范围高亮
- 操作按钮层级清晰，不互相争抢注意力
- 视觉重点始终围绕“当前能做什么”和“当前出了什么问题”
